Projecte Optativa: FitXarxa

Una cadena de centres esportius vol una aplicació per gestionar:

  • Entrenadors (amb foto de perfil)
  • Sessions / Classes dirigides (zumba, spinning, ioga…)
  • Centres (cada centre ofereix diverses sessions, i una mateixa sessió pot oferir-se en diversos centres)

L’aplicació tindrà dues parts:

  1. API REST en Laravel (servidor) per gestionar dades i servir JSON/fitxers.
  2. SPA en Angular (client) per consumir l’API i oferir una interfície d’administració.

1) Backend (Laravel) — FitXarxa API

1.1 Models, migracions i dades de prova

Crea models i taules mínimes:

  • Entrenador: nom, cognoms, imatge (nullable)
  • Sessio: titol, dataP (data), places (número), entrenador_id (FK)
    • Relació: Many-to-One (una sessió té un entrenador)
  • Centre: nom, adresa

Crea un Seeder (p. ex. FitXarxaSeeder) que insereixi com a mínim:

  • 1 entrenador
  • 2 sessions associades a aquest entrenador

1.2 Endpoints CRUD bàsics (JSON)

Implementa rutes a routes/api.php i mètodes a un ApiController:

  • GET
    • Llistar entrenadors i sessions
    • Consultar per id entrenadors i sessions
  • POST
    • Crear entrenador
    • Crear sessió
  • PUT
    • Modificar sessió
  • DELETE
    • Eliminar sessió
    • Eliminar entrenador

1.3 Centres ↔ Sessions (Many-to-Many)

Afegeix una relació Many-to-Many entre Centre i Sessio:

  • Crea la taula pivot centre_sessio amb:
    • centre_id, sessio_id
    • foreign keys amb onDelete('cascade')

Defineix belongsToMany() als models i crea endpoints per:

  • Associar una sessió a un centre (attach)
    • Exemple: centre-sessio/add/{idCentre}/{idSessio}
  • Desassociar una sessió d’un centre (detach)
    • Exemple: centre-sessio/remove/{idCentre}/{idSessio}
  • Cercar centres a partir de la id d’una sessió
    • Exemple: centres/cerca-per-sessio/{idSessio}
  • CRUD bàsic de centres:
    • Llistar, cercar per id, inserir

1.4 Imatge de l’entrenador (upload + endpoint de fitxer)

Afegeix el camp imatge (nullable) a entrenadors i desa a BD el nom del fitxer.

Implementa:

  • Endpoint POST per crear entrenador amb imatge
    • Mou el fitxer a public/uploads/imatges
    • Genera nom únic (p. ex. amb uniqid())
  • Endpoint POST per editar entrenador amb imatge
    • Es fa amb POST (no PUT) per facilitar l’enviament de fitxers
  • Endpoint GET per retornar la imatge
    • Ha de respondre amb el Content-Type correcte (p. ex. image/jpeg)
  • Defineix la ruta de la carpeta d’imatges com a variable d’entorn
    • Exemple: RUTA_IMATGES

1.5 Testing de l’API

  • Prova els GET amb navegador.
  • Prova POST/PUT/DELETE amb Postman (headers i body correctes, i upload de fitxer quan toqui).

2) Frontend (Angular) — FitXarxa Client

2.1 Estructura SPA + Bootstrap + components

Crea un projecte Angular i afegeix Bootstrap via CDN.

Crea components mínims:

  • nav-bar (navegació)
  • welcome (pàgina d’inici amb imatge a assets/)
  • entrenador-list
  • sessio-list

2.2 Rutes (SPA)

Configura l’AppRoutingModule i utilitza <router-outlet> per navegar entre:

  • Inici
  • Llistat d’entrenadors
  • Llistat de sessions
  • Formularis de crear/editar

2.3 Models (interfaces) i serveis (HttpClient)

Crea interfaces:

  • IEntrenador (id, nom, cognoms, imatge|null)
  • ISessio (id, titol, dataP, places, entrenador_id, entrenador: IEntrenador)

Crea serveis:

  • dadesEntrenadors (GET/POST/DELETE + POST d’edició amb FormData)
  • dadesSessions (GET/POST/PUT/DELETE)

Usa HttpClient, Observable i subscribe().

2.4 Consum real de l’API

Configura proxy.conf.json i angular.json per apuntar /api a l’API Laravel (http://localhost:8000).

2.5 Llistats amb dades reals

  • entrenador-list: mostra nom, cognoms i la imatge amb <img [src]="...">
  • Al backend: fes que el camp imatge retorni una URL de l’endpoint de la imatge (no només el nom del fitxer)
  • sessio-list: mostra dades de sessió i, a la columna “Entrenador”, mostra:
    • sessio.entrenador.nom i sessio.entrenador.cognoms
    • (Requereix eager loading al backend)

2.6 Filtre dinàmic (pipe + ngModel)

Afegeix un input de cerca i un pipe que filtri entrenadors per:

  • nom o cognoms
  • sense distingir majúscules/minúscules

2.7 Formularis (Reactive Forms) + CRUD complet

Crea components:

  • sessio-create i sessio-edit
  • entrenador-create i entrenador-edit

Requisits:

  • Valida camps i mostra errors a la vista
  • A sessió (create/edit), l’entrenador es tria amb desplegable carregat des del servei d’entrenadors
  • A entrenador (create/edit), usa FormData per enviar text + fitxer (imatge)
    • L’edició d’entrenador s’envia amb POST
  • Afegeix botons d’editar i eliminar a les llistes
    • Després d’eliminar, refresca la llista (p. ex. tornant a cridar ngOnInit())
